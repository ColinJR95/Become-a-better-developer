# Become-a-better-developer Tips for responsive design

Here are some useful tips for help your website become more responsive!

There is a difference between DEVICE INDEPEDENT PIXAL and HARDWARE PIXEL

Device independent pixal

  • This is the amount of pixels that you mobile browser will actually give you.
  • Will take up the same about of of space regardless of the pixel density of the display.
  • Usually half of the amount of hardware pixels

Hardware Pixels

  • This is the unit of pixel that is your hardware.(ex. Computer, phone, ipad)
  • We have to tell the broswer that mobile devices have different pixel limits
  • Prevent the browser from font boosting by Calculating Viewport Width

Calculating Viewport Width

  • To get the the Width (in dips) remember to dive by 1/2 dividing the number of physical pixels by the number of Device Pixel Ratio!
  • Remember that DPR is is the amount of Dips it will take per pixel

Setting the Viewpoint in your html code!

  • In order to set the view point we use a special code <meta name="viewport"content="width=device-width,initial-scale=1">
  • By setting initial-scale=1 we are telling the broswer to establish a 1 to 1 ratio between device independent pixels and and CSS pixels.
  • CSS pixels are what we all are used to working with most of the time
  • Setting it to a 1:1 ration we tell the browser to keep the page constant while rotating to landscape mode.
  • It is important to use relative units instead of absolute units "img, embed, object, video {}" max-width: 100%;
  • Finally, another tip to know is that fingers are about 40pixels wide so it is important to keep that in mind and when setting up for clickable links or buttons. It is reccomened that we use 48px wide by 48px tall.
  • Make sure that buttons are 40px apart from each other so it would look like this for a button <nav a, button { min-width: 48px; min-height: 48px;} >
  • Start off by thinking about and optimizing for smaller devices first, then move on to bigger devices like an ipad finally you can start optimizing for a laptop or desktop. This will allow us to prioritize the key content
